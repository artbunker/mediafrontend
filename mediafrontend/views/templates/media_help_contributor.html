Semantic tags
Some tags have a special meaning that can affect how a medium's view page and thumbnail are displayed.
#set:{set name} Used to create visual set groupings below a medium on the view page.

{set name} can be any text string but should probably be something you can remember and re-type if you're adding it to multiple media manually.

By default the order that set media appear in the set is by their Creation date (not their Upload date) from earliest to latest, but this can be overridden by adding :{weight} to the set tag after {set name} (e.g. #set:some set name:5) in which case the medium with {weight} will appear after any media without {weight} and be sorted among any other media with {weight} from least to greatest.

If two media in a set have the same {weight} then they will be sorted by their respective Creation date from earliest to latest

A medium can be part of multiple sets, in which case multiple set containers will appear below the medium.
#cover:{medium ID} Used to make the thumbnail of another medium appear for the medium you've added the tag to.

This is mostly useful for audio/text/archive/application media, which normally only appear as a filetype placeholder icon.

In addition to displaying a thumbnail in search results and in set lists, for audio the cover medium will appear above the actual player on the view page.
#render:{html fragment{# | markdown#}} Used to embed text types directly into the view page
#next:{medium ID} Used to specify another medium as the next in a sequence.
#prev:{medium ID} Used to specify another medium as the previous in a sequence.
#disable:{feature name} Used to disable a normal feature of the view page's interactivity. E.g. adding comments, or placing stickers. But should leave any interactions that already happened in place.
#suppress:{feature name} Used to suppress a normal feature of the view page. E.g. the comment field, or stickers
#mirror:{URI} Used to specify a mirror of a medium at an outside URI
#superior of:{medium ID} Used to indicate a medium is a superior version of another medium
#inferior of:{medium ID} Used to indicate when a medium is an inferior version of another medium
#blurb:{text} Used to display some short plaintext below the info bar on the view medium page
#text:plain Used to display the actual text contents of a medium on the view medium page (escaped so that markup won't affect its display)
#text:html fragment Used to embed the actual text contents of a medium on the view medium page (unescaped)
#embed:{youtube | vimeo}:{video_id} Used to embed an external video on the view medium page
#link:{URI} Used to override default view page behavior where a medium summary links to the original file, and instead force it to link to the specified URI
